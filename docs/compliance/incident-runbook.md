# Incident Runbook

Version: `2026-04-02`

## Trigger
- Suspected data breach
- Unauthorised account access
- Exposure of community/health-related content
- Misconfigured robots/access control

## First 72 hours
1. Contain access and preserve evidence.
2. Classify affected systems, users, and data categories.
3. Determine whether special-category health data is involved.
4. Decide whether supervisory authority notification is required under GDPR Art. 33.
5. Decide whether user communication is required under GDPR Art. 34.
6. Document timeline, decisions, and remediation.

## Minimum records
- Incident start time
- Reporter
- Systems affected
- Data types affected
- Containment action
- Notification decision
- Corrective action owner
